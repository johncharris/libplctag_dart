## What this project is

A Dart FFI wrapper around the native [libplctag](https://github.com/libplctag/libplctag) C library, modelled after [libplctag.NET](https://github.com/libplctag/libplctag.NET). It lets Dart/Flutter applications communicate with industrial PLCs over Allen-Bradley EIP and Modbus TCP. Status: early-stage prototype, not production-ready.

Bundled native libraries (no system install required):
- `lib/libplctag/windows/x64/plctag.dll`
- `lib/libplctag/linux/x64/libplctag.so`
- `lib/libplctag/linux/aarch64/libplctag.so`

---

## Architecture (top → bottom)

```
Tag<T>              lib/tag_of_t.dart           type-safe generic API; delegates to Tag via IPlcMapper<T>
Tag                 lib/tag.dart                untyped facade; ~50 properties and raw data accessors
NativeTagWrapper    lib/native_tag_wrapper.dart core state machine; assembles attribute strings; owns tag handle
NativeTag           lib/native_tag.dart         thin delegate to plctag.* static methods
plctag              lib/native/plctag.dart      lazy-loads native library; converts Dart → FFI types
NativeLibrary       lib/native/generated_bindings.dart   AUTO-GENERATED by ffigen — do not edit
plctag.dll/.so      lib/libplctag/...           bundled C library
```

`NativeTagBase` (`lib/native_tag_base.dart`) is the abstract interface between `NativeTagWrapper` and `NativeTag`. This is the seam for mocking in tests.

Tag lifecycle enforced by `NativeTagWrapper`: **create → initialize() → read()/write() → dispose()**. Calling in the wrong order throws.

## Feature gaps vs libplctag.NET

### Missing data type mappers
All four unsigned integer types are in the C API (`libplctag.h`) and in `generated_bindings.dart` but have no mapper or `NativeTagWrapper`/`Tag` accessors:

| Mapper | PLC type | C accessor |
|--------|----------|------------|
| `UsintPlcMapper` | USINT | `get/set_uint8` |
| `UintPlcMapper` | UINT | `get/set_uint16` |
| `UdintPlcMapper` | UDINT | `get/set_uint32` |
| `UlintPlcMapper` | ULINT | `get/set_uint64` |

### C API functions not yet wired up
These exist in `libplctag.h` and `generated_bindings.dart` but are not exposed through the `NativeTagBase → NativeTag → plctag → NativeTagWrapper → Tag` chain:

- `plc_tag_shutdown` — needed for clean teardown
- `plc_tag_check_lib_version` — version guard at startup
- `plc_tag_get_int_attribute` / `plc_tag_set_int_attribute` — runtime attribute mutation
- `plc_tag_register_callback` / `plc_tag_unregister_callback` — event callbacks (commented out in `NativeTag` and `plctag.dart`)
- `plc_tag_register_logger` / `plc_tag_unregister_logger` — logger hook (commented out)

### Incomplete implementations
- **Multi-dimensional arrays**: 2D/3D decode/encode paths are commented out in `lib/data_types/plc_mapper_base.dart` and `lib/data_types/bool_plc_mapper.dart`. The 1D path works.
- **`Tag<T>` element count**: `arrayDimensions` is stored but `getElementCount()` is not applied to `NativeTagWrapper.elementCount` during `initialize()`.
- **Async operations**: `TIMEOUT_VALUE_THAT_INDICATES_ASYNC_OPERATION` and `ASYNC_STATUS_POLL_INTERVAL` constants exist in `NativeTagWrapper` but no `readAsync()`/`writeAsync()` methods are implemented.
- **`StringPlcMapper` Micro800 element size**: Set to 256 with a "To be Confirmed" comment.
- **Library extraction**: The Windows/Linux path logic in `lib/native/library_extractor.dart` that locates bundled binaries is commented out; currently relies on the library being on the system path.

### Tests
`test/run_test.dart` contains only placeholder Dart stdlib tests (`String.split`, `int.remainder`) — zero coverage of this library.

## Implementation plan

### Phase 1 — Bug fixes (do first)
Fix the 5 confirmed bugs listed above. These affect correctness and API usability.

### Phase 2 — Expose full C API
Wire each missing function through the `NativeTagBase → NativeTag → plctag → NativeTagWrapper → Tag` chain. Mechanical work: add an abstract method to `NativeTagBase`, implement in `NativeTag`, add a static method in `plctag`, expose in `NativeTagWrapper`/`Tag`. The unsigned accessor functions (`get/set_uint8/16/32/64`) also need to be added to `Tag` as `getUInt8(offset)` etc. and wired down to `plctag`.

Uncomment and complete the callback functions. The Dart callback signature will need an `NativeCallable` or `Pointer<NativeFunction<...>>` — see `dart:ffi` docs for passing Dart functions to C.

### Phase 3 — Complete data type coverage
Add the four unsigned mappers by copying the signed equivalents (`sint`, `int`, `dint`, `lint`) and swapping the accessor calls. Complete multi-dimensional array paths in `PlcMapperBase`, then verify all other mappers inherit it correctly.

### Phase 4 — Async operations
`NativeTagWrapper.readAsync()`: call `plc_tag_read(handle, 0)` (timeout=0 → returns `PENDING`), then poll `plc_tag_status` every `ASYNC_STATUS_POLL_INTERVAL` ms using `Future.delayed` until status is `Ok` or an error. Surface as `Future<void>`.

For event-driven async, register a Dart `NativeCallable.listener` via `plc_tag_register_callback` and expose a `Stream<TagEvent>` on `Tag<T>` with typed events (`ReadCompleted`, `WriteCompleted`, `Aborted`, `Destroyed`).

### Phase 5 — Testing
**Unit tests (no PLC required):**  
Create `MockNativeTag implements NativeTagBase` backed by an in-memory byte buffer. Use it to test:
- `NativeTagWrapper` state machine (lifecycle violations, dispose idempotency)
- Attribute string generation (each property → correct key/value pair)
- All mapper encode/decode round-trips for scalar and array forms
- Bool mapper PLC-type variants (Omron vs ControlLogix)
- `TagInfoPlcMapper` against known byte sequences
- `Status.fromInt` for all 40 codes

**Integration tests (opt-in):**  
Guard with an environment variable (`PLC_GATEWAY`). When set, run round-trip read/write, async flow, callback ordering, and tag listing against a real PLC or the libplctag software simulator.

### Phase 6 — Package polish
- Update `pubspec.yaml` SDK to `>=3.0.0 <4.0.0` and bump all dependencies
- Restore library extraction path logic in `library_extractor.dart`
- Call `plc_tag_check_lib_version` in `plctag._extractLibraryIfRequired()` to fail fast on version mismatch
- Fix package exports in `libplctag_dart.dart`
