## Project Overview

Dart FFI bindings for [libplctag](https://github.com/libplctag/libplctag), enabling Dart/Flutter apps to communicate with industrial PLCs (Allen-Bradley EIP and Modbus TCP). This is an early-stage port of libplctag.NET and is not production-ready.

## Commands

```bash
# Install dependencies
flutter pub get

# Run tests
dart test

# Analyze / lint
dart analyze

# Regenerate FFI bindings from libplctag.h (after C header changes)
flutter pub run ffigen

# Run example Flutter app
cd example && flutter run
```

Run a single test file:
```bash
dart test test/run_test.dart
```

## Architecture

The project uses a strict layered architecture. Each layer only communicates with the one directly below it:

```
Tag<T>  (lib/tag_of_t.dart)       — type-safe generic API; uses IPlcMapper<T> for encode/decode
  ↓
Tag     (lib/tag.dart)             — untyped tag; offset-based read/write of raw PLC data
  ↓
NativeTagWrapper (lib/native_tag_wrapper.dart)  — tag lifecycle, attribute string generation, all property state
  ↓
NativeTag (lib/native_tag.dart)    — thin delegate to plctag.* static methods
  ↓
plctag  (lib/native/plctag.dart)   — lazy-loads native library; converts Dart types to FFI types
  ↓
NativeLibrary (lib/native/generated_bindings.dart)  — AUTO-GENERATED by ffigen; do not edit manually
  ↓
plctag.dll / libplctag.so          — pre-compiled C library (lib/libplctag/{windows,linux}/...)
```

**Tag lifecycle** (enforced in NativeTagWrapper): create → `initialize()` → `read()`/`write()` → `dispose()`. Operations outside this order will fail.

## Key Patterns

### Type Mappers
Adding a new PLC data type means implementing `IPlcMapper<T>` (lib/data_types/iplc_mapper.dart). `PlcMapperBase<T>` (plc_mapper_base.dart) handles the scalar case. The mapper handles both encoding (Dart → PLC bytes) and decoding (PLC bytes → Dart), including byte-order differences between PLC families.

### FFI Bindings Regeneration
`lib/native/generated_bindings.dart` is generated from `libplctag.h` via `ffigen`. The output path and header entry-point are configured in `pubspec.yaml` under the `ffigen:` key. Only regenerate when the C header changes.

### Attribute Strings
The native library is configured via an ampersand-delimited attribute string (e.g. `protocol=ab_eip&gateway=192.168.1.1&name=MyTag&...`). This string is assembled in `NativeTagWrapper.getAttributeString()` from all the tag's properties.

### String↔FFI Conversion
`lib/native/extensions.dart` provides `String.toInt8()` and related helpers that allocate native memory for C strings. All FFI pointer allocations must be freed after use.

### Native Library Loading
`LibraryExtractor` (lib/native/library_extractor.dart) locates and loads the platform-specific binary at runtime. Bundled binaries live under `lib/libplctag/`.
